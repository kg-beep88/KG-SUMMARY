KG STOCK + INTERNAL CALENDAR — EASY START
========================================

THIS VERSION HAS NO GOOGLE CALENDAR.
No service account. No JSON key. No Edge Function. No Cron.

EXISTING WEBSITE UPGRADE
------------------------
1. Keep your current supabase-config.js.
2. Upload all other website files to the GitHub repository.
3. Replace the old files.
4. Delete calendar-config.js from GitHub if it is still there.
5. Press Ctrl + F5 on the website.
6. Sign in and open Calendar.

USE THE CALENDAR
----------------
1. Click Calendar.
2. Click a date.
3. Press + Add work.
4. Choose the worksite, address, PIC and dates.
5. Press Save site work.
6. Click an existing calendar task to edit it.

NEW SUPABASE PROJECT ONLY
-------------------------
1. Put your Supabase Project URL and publishable key into supabase-config.js.
2. Run supabase/sql/01-database-setup.sql in Supabase SQL Editor.
3. Enable Google login in Supabase Authentication. This login is only for access; it does not read Google Calendar.
4. Add your GitHub Pages URL to Supabase Auth URL Configuration.
5. Upload all files to GitHub Pages.

OPTIONAL CLEANUP
----------------
If the old version created Google Calendar Cron objects, run:
supabase/sql/03-optional-remove-old-google-calendar.sql

You may delete sync-google-calendar from Supabase Edge Functions, but it is not necessary.
